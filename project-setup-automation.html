Pre-flight — 7 Required Fields on the Sales Task

  • Contact Name(s) — who the project is for
  • Email PRIMARY — client contact email
  • Contact Number — phone for site team
  • Deal Value (incl GST) — sets progress claim amounts
  • Job Address — site location
  • Construction Plans — linked drawings
  • Approved Proposal — signed scope of works

Phase 1 — Trigger

MAKE.COM INSTANT
Fires automatically when task status → Closed Won • ~2 seconds
  1. Sales task moves to "Closed Won" in ClickUp
    Anyone on the sales team (Gemma, Michael, etc.) moves the task status. A ClickUp API webhook fires instantly to Make.com — filtered to only taskStatusUpdated events where the new status is "closed won".
  2. Make.com checks all 7 required fields exist
    Router with pre-flight filter checks the sales task's custom fields:

    FieldWhy Required
    Contact NameFolder naming, project doc
    EmailClient contact, project doc
    PhoneSite team needs this
    Deal ValueProgress claim amounts
    Job AddressProject doc, site reference
    Construction PlansPlans reference table
    Approved ProposalScope extraction, site tasks
    If ANY field is missing → Route B fires (no folder created, "fix these fields" comment posted).
  3. Project folder created from template (10 lists)
    Only runs if all 7 fields pass (Route A).

    Make.com calls POST /v2/space/54603442/folder_template/t-90161746266 with return_immediately: true. ClickUp starts provisioning the folder in the background — 10 template lists, all template tasks, custom fields, and the Project Doc.

    Provisioning takes 2–3 minutes. By the time you run the orchestrator, it's usually done.
  4. Assigned comment posted to Michael
    An assigned comment appears in Michael's ClickUp tray:

    Project folder created for TS260306 Sunshine Kebabs.

    Run in Claude Code:
    /project-setup-orchestrator 86d2xxxxx

    Folder ID: 90169xxxxxx

    If fields were missing, the comment says:
    Project Setup Blocked — Required fields missing.
    Please check and complete: Contact Name, Email, Phone, Deal Value, Job Address, Plans, Proposal.
    Move task back to Won to re-trigger.

Phase 2 — Orchestrator

CLAUDE CODE
Run by Michael locally • /project-setup-orchestrator {taskId}
  1. Find the project folder (already created by Make.com)
    The orchestrator searches SBI PROJECTS space for a folder matching the sales task's job number prefix (e.g. 260306). Make.com already created it — the orchestrator finds it, doesn't create a duplicate.
  2. Poll for provisioning readiness (smart check)
    No fixed sleep timers. The orchestrator polls every 10 seconds:

    • Fetch GET /v2/folder/{id}/list
    • Count lists — need all 10
    • Check Project Doc exists
    • Up to 3 minutes (18 attempts)

    If already provisioned (usual case), continues instantly. If still in progress, waits only as long as needed.
  3. Detect project type — BUILD or DESIGN
    Checked in order:
    1. Task/folder name contains "DESIGN" or "CERT & DESIGN" → DESIGN
    2. Task/folder name contains "BUILD" → BUILD
    3. Job Type custom field on sales task
    4. Ask user if still unclear
  4. Read sales documents and custom fields
    Reads from the sales task: contact details, deal value, job address, plans links, proposal, Buildsoft, date sold, notes.

    Also checks the shared drive for uploaded documents at:
    G:\Shared drives\SBI JOB FILES_Share\{project}\
  5. Check what's already done — skip completed steps
  6. Call sub-skills in sequence (Phase 3)

Phase 3 — Sub-Skills Chain

CLAUDE CODE SEQUENTIAL
Run by Claude automatically • 4 sub-skills in order
  1. Sales Handover — populate folder with sales data
    Sub-skill: /sales-handover

    • Folder renamed: 260306 BUILD Sunshine Kebabs
    • Sales task multi-listed to List 01 + prefixed "SOLD"
    • Template tasks customised with job name
    • Project Info Doc populated (client, plans, address)
    • Progress claims created in List 04 (40/30/20/10% BUILD, 60/40% DESIGN)
    • Schedule dates set in List 06 (sequential from date sold)
    • Site works tasks created from proposal scope
    • Handover task assigned (Chadd, Gemma, Gail, Michael)
  2. Budget Cost Sheet — generate Excel from Buildsoft
    Sub-skill: /budget-cost-sheet • BUILD only

    • Reads Buildsoft Excel from 1.2 COST SHEETS/
    • Pre-flight: verifies pricing data exists (Qty × Rate × Total > 0)
    • Generates "SBI Cost Budget" tab with Detail + Summary tables
    • Styled PDF export with two donut charts (Cost Breakdown + Sales Breakdown)
    • Totals verified against deal value
  3. Budget to Procurement — create procurement tasks
    Sub-skill: /budget-to-procurement

    • Reads cost codes from SBI Cost Budget tab
    • One task per cost code in List 08 (e.g. 310 Sunshine Kebabs BUILD Materials)
    • Budget Allowance SBI field set on each task
    • Procurement category + workflow initialised
    • Existing tasks matched by code — no duplicates on re-run
  4. Project Dashboard — generate & deploy
    Sub-skill: /project-dashboard

    • Fetches all project data from ClickUp (claims, budget, schedule, procurement, compliance)
    • Generates interactive HTML dashboard with 7 tabs
    • Deployed to GitHub Pages: michael-sbi.github.io/sbi-dashboards/{slug}/
    • Saved to shared drive
    • Link posted to Projects Meeting task (Tuesday review)
